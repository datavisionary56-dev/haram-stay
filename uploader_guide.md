# دليل رفع الفنادق إلى HaramStay (نظام الأتمتة)

هذا الدليل يشرح كيفية رفع الفنادق الجديدة وتحديث البيانات تلقائياً باستخدام سكريبتات بايثون، دون الحاجة للإدخال اليدوي.

## المتطلبات المسبقة
- تأكد من تثبيت Python.
- تأكد من وجود مكتبات Firebase المطلوبة (`firebase-admin`).

## الخطوات

### 1. تحضير الصور
- ضع مجلدات الفنادق في المسار التالي على جهازك: `Desktop/My_Hotels`.
- تأكد أن كل مجلد يحمل اسم الفندق بالإنجليزية (مثلاً: `dar_al_wafideen`).
- ضع صور الفندق داخل مجلد اسمه (نفس اسم الفندق) لتجنب التكرار، أو مباشرة داخل المجلد الرئيسي للفندق.

### 2. تشغيل المعالجة المحلية (Local Processor)
هذا السكريبت يقوم بمسح الصور وتجهيزها للرفع.
```bash
python local_processor.py
```
- سيقوم السكريبت بإنشاء ملف `metadata.json` في كل مجلد يحتوي على أسماء الصور.

### 3. رفع البيانات والصور (Sync DB)
هذا هو السكريبت الرئيسي الذي يقوم بكل العمل:
- يرفع الصور إلى مجلد `public/images` في المشروع.
- يحدث قاعدة بيانات Firestore ببيانات الفندق (الاسم، السعر، الموقع).
- يربط الصور بالفندق تلقائياً.

لتشغيله، افتح التيرمينال ونفذ:
```bash
python sync_db.py
```

### ملاحظات هامة
- **تعديل الأسعار والموقع**: يمكنك تعديل البيانات الافتراضية (مثل السعر 300 ريال) مباشرة داخل ملف `sync_db.py` في قسم `HOTEL_DATA_MAP` قبل التشغيل.
- **تحديث الواجهة**: بعد تشغيل السكريبت، قد تحتاج لتحديث الصفحة في المتصفح لرؤية التغييرات.

### هيكلة البيانات (Data Lock)
تم تثبيت البيانات التالية لفندق دار الوافدين كمرجع أساسي:
- **الاسم**: فندق دار الوافدين
- **السعر**: 300 ريال
- **المسافة**: 800 متر
- **الموقع**: شارع إبراهيم الخليل
